Fotobox Qt WebEngine Host

Local demo page for Linux / Lokale Demo-Seite für Linux

Deutsch

Diese App ist der Linux-Port deines Windows-WebView2-Hosts, jetzt mit Qt WebEngine. Qt WebEngine basiert auf Chromium, ist aber nicht der Google-Chrome-Browser selbst. Die Webansicht lädt HTML, CSS und JavaScript in einem nativen Linux-Fenster.

Die JavaScript-Schnittstelle wird vom Host automatisch in jede geladene Seite injiziert. Du musst also in deiner eigenen HTML-Datei keine externe Quelle einbinden, solange die Seite in dieser App läuft.

Wichtiger Hinweis zu lokalen Dateien

.html, .htm und ähnliche Dateien können direkt lokal geladen werden. Eine .php-Datei wird als Datei geladen, aber nicht als PHP ausgeführt. Für PHP brauchst du weiterhin eine laufende URL wie http://127.0.0.1:8080/index.php.

English

This app is the Linux port of your Windows WebView2 host, now using Qt WebEngine. Qt WebEngine is based on Chromium, but it is not the Google Chrome browser itself. The embedded view loads HTML, CSS, and JavaScript in a native Linux window.

The JavaScript bridge is injected automatically by the host into every loaded page. That means your own HTML page does not need to include any external source as long as it is running inside this app.

init.json explained

KeyMeaningExample
url Preferred startup target. It can be a full URL, a local URL without scheme, or a local file path relative to init.json. "url": "127.0.0.1:8080/app"
"url": "pages/index.html"
defaultUrl Fallback base URL if url is empty and localIndexPath could not be resolved. "defaultUrl": "http://127.0.0.1"
defaultPort Port appended to defaultUrl. "defaultPort": 8080
localIndexPath Local HTML file relative to init.json. Used when url is empty or cannot be resolved. "localIndexPath": "wwwroot/index.html"
kiosk Starts frameless and fullscreen when set to true. "kiosk": true
title Optional window title. If empty, the hardcoded fallback title is used. "title": "My Photo Booth"
icon Optional icon path relative to init.json. PNG is usually the simplest choice on Linux. "icon": "assets/app.png"
allowDevTools Allows F12 or Ctrl+Shift+I to open Chromium DevTools in a separate window. "allowDevTools": true

Important note about local files

.html, .htm, and similar files can be loaded directly. A .php file can be loaded as a file, but it will not execute PHP. For PHP, you still need a running URL such as http://127.0.0.1:8080/index.php.

JavaScript bridge examples / JavaScript-Schnittstelle

These examples are complete and can almost be copied 1:1 into your page.

function whenHostAppReady(timeoutMs = 5000) { return new Promise((resolve, reject) => { if (window.hostApp) { resolve(window.hostApp); return; } const timer = setTimeout(() => { reject(new Error('hostApp bridge not ready')); }, timeoutMs); window.addEventListener('hostAppReady', () => { clearTimeout(timer); resolve(window.hostApp); }, { once: true }); }); } async function minimizeApp() { const hostApp = await whenHostAppReady(); hostApp.minimize(); } async function maximizeApp() { const hostApp = await whenHostAppReady(); hostApp.maximize(); } async function restoreApp() { const hostApp = await whenHostAppReady(); hostApp.restore(); } async function setKiosk(on) { const hostApp = await whenHostAppReady(); hostApp.setKiosk(!!on); } async function closeApp() { const hostApp = await whenHostAppReady(); hostApp.close(); }
